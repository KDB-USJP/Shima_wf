🏝️ Shima Troubleshooting

Maintenance and Fixes for the Discerning AI Artist

🔍 "Cannot validate output" / Missing Models

This is the most common error. It means ComfyUI cannot find a model, Lora, or VAE referenced in your workflow.

  • Action: Run the Shima Workflow Checker node. It will highlight specifically which asset is missing.
  • Fix: Ensure your models are in ComfyUI/models/checkpoints/ and LoRAs are in ComfyUI/models/loras/.

🌐 Black Screens / Connection Refused (Cloud PC)

If the Styler, Excel Editor, or Gallery pages show a "Connection Refused" or a black screen on platforms like MimicPC:

  • Protocol: Ensure you are accessing your ComfyUI via https if required, or check if the port mapping is active.
  • Pathing: Some cloud proxies require a trailing slash: use .../shima/styler/ instead of .../shima/styler.
  • Browser: Try a Hard Refresh (Ctrl + F5). If that fails, check the browser console for "Blocked by X-Frame-Options".
  • Path: Note that help is now at /shima/shima_help.html

⚡ Use Everywhere Chain Issues

Use Everywhere (UE) is the "Magic Wire" system. Sometimes the magic gets crossed.

  • Group Restrictions: Shima adds group_regex restrictions to nodes when you place them within an Island Group. This keeps wires from leaping to other islands.
  • The Fix: If a workflow fails to connect after you've moved nodes around, use the Shima Custodian and click "Clear Group Restrictions" to reset the wiring state.
  • Override Shield: Remember that "Manual Overrides" (Model/VAE/Clip) are shielded from UE by default to prevent recursion loops. Connect these manually!
