# Shima v2.0.5: Registry Recovery & Dependency Restoration 🏝️📦

A maintenance release focused on restoring core dependency management functionality and reinforcing asset security.

## 🚀 Key Improvements

### 1. Smart Dependency Detection
- **Upgraded Scanner**: The Dependency Generator now uses a "Greedy" detection model. Instead of looking for specific nodes, it scans every widget for file extensions (`.safetensors`, `.ckpt`, `.pt`, etc.) and keywords (`image_path`, `model`, `vae`).
- **Comprehensive Coverage**: This fix ensures that nodes like **Shima Interactive Crop** and other custom inputs are correctly identified and listed in the dependencies.
- **Python Registration**: Restored node registration in the core module.

### 2. Asset Integrity (AssetManager v2)
- Implemented **Dynamic Size Verification** using `HEAD` requests to verify remote assets before downloading.
- Improved sandboxing for custom `save_path` overrides to prevent directory traversal.

### 3. Documentation & Branding
- Updated **Shima Node Reference** with full descriptions for the dependency system.
- Version bumped across all core and public modules to `2.0.5`.
