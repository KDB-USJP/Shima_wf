# Shima v2.0.6: Sticker Systems & Tiny Scaling 🏝️🎨

This release introduces major quality-of-life improvements to the Sticker and Noodman nodes, focusing on robust UI interactions, proportional resizing, and a new ultra-compact "Icon Mode".

## 🚀 Key Improvements

### 1. Robust Sticker Management
- **Aggressive Widget Cleanup**: All widgets on the Sticker node are now hidden by default using a proactive cleanup loop. This prevents visual clutter and accidental interaction with background parameters.
- **Double-Click & Right-Click Access**: Restored the double-click trigger for the configuration modal. Additionally, added a standard "Shima: Configure Sticker..." option to the node's right-click context menu for improved accessibility.
- **Settings Modal Hardening**: Refactored the settings dialog to be entirely independent of hidden widgets, preventing silent crashes when opening the menu.

### 2. Smart Resizing & Proportionality
- **Proportional Dragging**: Sticker and Noodman nodes now automatically maintain their aspect ratio during manual resizing. No more squashed logos!
- **Reload Persistence**: Enhanced the aspect-ratio logic to prevent "snapping" during workflow reloads. User-defined sizes are now respected regardless of image loading speeds.

### 3. Icon Mode (Ultra-Compact)
- **20px Minimum Bounds**: Reduced the hard minimum size constraint from 50px to **20px**.
- **Padding Optimization**: Reduced internal padding to 10px, allowing Stickers and Noodman to clear-render even at tiny, icon-sized scales.
- **Silver Bullet Fix**: Explicitly overrode LiteGraph's default 80px minimum size for these utility nodes.
